CodeViz在Ubuntu12.04下安装部署总结

CodeViz是一款在C/C++源代码中分析函数之间调用关系的开源工具。原理是给GCC打个patch，让它在编译C/C++文件时导出一个函数流图，最后用graphviz来进行图形的绘制。

今天折腾了下CodeViz在Ubuntu12.04下的安装部署，还是遇到了很多的问题，不过幸好最后还是解决了。

CodeViz版本：1.0.12
该版本只支持GCC 4.6.2

  1. 首先安装graphviz
    在ubuntu下安装graphviz只需要使用apt-get命令即可
    sudo apt-get install graphviz

  2. 然后安装Codeviz，这个从github上能够下下来，原网址已经失效.

  3. 将其放在某个目录下，进行解压。这个1.0.12版本的graphviz只支持GCC 4.6.2，那么从gnu的官网上可以下到GCC 4.6.2

  4. GCC 4.6.2下下来以后，将其放在codeviz/compilers的目录下，然后运行install_gcc-4.6.2.sh,这步会将初始的gcc 4.6.2打上patch，并将其安装到某个目录，从sh脚本来看，安装的目录为 $HOME/gcc-graph，所以我们 echo $HOME就能知道最后整个patch过的gcc安装在了哪个目录下。

这一步的问题是最多最复杂的，我碰到了以下的几个问题

(1) 首先是出现 gcc configure: error: Building GCC requires GMP 4.2+, MPFR 2.3.1+ and MPC 0.8.0+
GCC的编译需要GMP, MPFR以及MPC三个库的支持，我们不需要自己去寻找这些库下载，在目录 codeviz/compilers/gcc-graph/gcc-4.6.2/contrib里面有一个download_prerequisites 的脚本，这几个脚本的作用就是将GMP,MPC以及MPFR下载下来解压，并创建软链接，默认的这些文件都会在contrib目录中，而在codeviz/compilers目录中再次运行install_gcc-4.6.2.sh还是出现同样的错误，说明它并没找到这些依赖库，网上有看到可以解决的一些办法，最简单的就是在 covdeviz/compilers/gcc-graph/gcc-4.6.2 目录下建立对GMP,MPFR以及MPC的软链接，具体的命令为(默认在codeviz/compilers/gcc-graph/gcc-4.6.2的目录下)
ln -s ./contrib/gmp-4.3.2 ./gmp

ln -s ./contrib/mpfr-2.4.2 ./mpfr

ln -s ./contrib/mpc-0.8.1 ./mpc

(2) 第二个问题是 fatal error: bits/predefs.h: No such file or directory,从stackoverflow上找到了解决方法，安装gcc-multilib即可。

(3) 第三个问题出现的是 cannot find crt1.o，这也是在stackoverflow上找到了解决方案，我们用locate crti.o进行定位，发现它在/usr/lib/x86_64-linux-gnu中，方法就是将其软链接到/usr/lib64中，done.

(4) 第四个问题出现的是 .libs/alloc.o：could not read symbols：Bad value，解决方法在这里找到了，即对install_gcc-4.6.2.sh进行修改，将

../gcc-4.6.2/configure --prefix=$INSTALL_PATH --enable-shared --enable-languages=c,c++ || exit
make bootstrap

修改为
CFLAGS="-O2 -fPIC" ../gcc-4.6.2/configure --prefix=$INSTALL_PATH --disable-bootstrap --enable-shared --enable-languages=c,c++ || exit
make -j 4

解决上述问题后，再次运行 install_gcc-4.6.2.sh即可

5.接下去的步骤基本就比较简单了，这里略去。
